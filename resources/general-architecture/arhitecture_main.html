First Module - Account management

One of our most important function, is to provide a clean and user friendly registration and login functionalities. The user data will be saved in the firebase database.

The log in form has 4 main sections, fully developed. The first section is for filling the username and password linked to an account. The second section is another type of log in form but with the data from the google account linked for the user. The third, will enable the change password functionality. The last section will redirect the user to a register account form.

Second Module - Game management

Our second module, or core functionality is to allow the user to access real-time statictis about the in-games and access a list of games to play. The use of third-party APIs will be essential for this functionality to receive all the hero data and images. The main functionalities of this core module are displayed below:

  • The game must be played on most popular native display resolution;
  • UI must be controlled with a mouse/touchscreen interface
  • The game needs to be played in two players or player-computer format.
  • The game needs to have different dificulties that can be played.
  • In the game, information about the hero characters must be displayed.

    Input/Output data formats


    The data sent and receive between the client and the server is of various formats. The data transferred between the server and the user is protected using different protocols and encryptions. The user is able to send and receive plain data in the html pages. The data exchanged between the server and the external retailers will be in Json format due to Json high customizations and flexibility. The third party APIs (such as Marvel API) will offer certain web services that provide data in many formats, but mainly XML or Json for larger transactions and data imports.
