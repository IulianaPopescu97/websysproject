Abstract

MarvelHeroJam application is a gaming web platform intended for use by anyone wanting to develop their strategic thinking and hand-eye coordonation. On the website, players can enjoy a variety of small strategy games involving their favorite Marvel characters.

This selection of games is intended to give people a relaxing and entertaing way of enjoing their free time with their beloved heroes as well as having the opportunity of impoving skills that might be usefull in varied real-world scenarios.


Introduction

While developing this web platform, we have taken into consideration various aspects, such as the user interaction and gameplay and also the overall design and look of the website, which we considered vital for it's charm and succes.

As far as the design goes, we considerd that it would be the best fit for our webiste if we went for a uniform, simple and yet attracting interface that sported a selection of simple but eye-caching animations. When it comes to the color pallet, we tried to make the most out of the color sported by Marvel's superheros. Our goal is to provide the players on the platfom with a colorfoul and memorable interface that would keep them interested in trying out all of it's futures and functionalities. The design at it's core is build around the "Superhero" craze that Marvel created.

The user interaction is designed to be consistent, functional and also engaging. The user is meant to feel familiar, in control, but also is encouraged to discover on his own.


Front End Technologies

In the development of the user interface, we decided on using the Angular Framework, as it is a very popular choice in designing modern web application and it allows us to build single page application that look and feel up-to-date, reuse components and make good use of many already existing libraries and UI elements.

In our case, we used Angular version 10.1 for our angular server and NodeJS version 14.15.1 as our the package manger npm, as it facilitates an easy package install solution through it's package.json file. The versions we chose are the latest stable version and we chose them as they have LTS(Long Term Support) and have been pached with the latest security and functionality fixes.

In addition to this, we used a series of additional packages that helped acheve the level of functionality and modern look that we aimed for:

  • PrimeNg

    This package contains a multitude of pre-made modern UI elements like: buttons, drop-down lists, tables, etc. which helps us create a page faster and with our desired look.

  • Angular Material

    This package is very popular while developing with the Angular framework, as it suppliments the the native HTML UI elements and also the ones from missing from other packages like PrimeNg.

  • Font Awesome

    It includes a large collection of graphical elements that can be imported into our project, rather than manually recreating them in css, making our work a lot easier and faster. In our project, it is mostly used in order to add suggestive and modern icons to UI elements.

  • RxJS

    RxJS is a library for composing asynchronous and event-based programs by using observable sequences. It provides one core type, the Observable, satellite types (Observer, Schedulers, Subjects) and operators inspired by Array#extras (map, filter, reduce, every, etc) to allow handling asynchronous events as collections.

  • Bootstrap

    Bootstrap allows us to use pre-made templates for CSS classes and also graphical control elements like a modal, navbar etc.


Back End Technologies

For the Back End implementation, our plan is to use a Backend-as-a-Service platform that works with the REST API princile, in order to manage data to and from the client in the JSON format as to to facilitate a faster development. For this purpose we chose FireBase as our primary Back-End soution.

FireBase is a compllex BaaS that will also handle our data persistance needs, as it would serve us as an API, Server and also data storage solution.


API

The web service that we chose to integrate into our application is one for data in the JSON format, as well as images. The service that we use is called The Marvel Comics API which offers a large database of information and images related to the Marvel comics.

  • The Marvel Comics API is a RESTful service which provides methods for accessing specific resources at canonical URLs and for searching and filtering sets of resources by various criteria. All representations are encoded as JSON objects.
  • It allows us to request specific data object or collections that we are interested in.
  • For the requested image the API provides different query parameters that can be used to alter the images in various ways.

User interaction and design

The web application we propose consists of multiple components, as follows:


Authentication


For authentication we decided to make use of simple forms for both the registration and the login functionalities. This ensures that it would be easy to login as well as sign up for our service.

We tried to keep the registration form as small and not-intrusive as possible, as to not endager any important personal imformation to a possible leak or risk that a user would leave our page due to the fear of providing us with it's important privare details. The required fields in our sign-up form are: username, email and password.


Home page


The home page contains a list with the recommended games based on their rating that you can try out and play, a setting section where you can change you username or password and also the sign-out functionality.

Also, it contains a list of tips and tricks on how to play these games and achive the best results.


Games page

On this page the logged-in users can enjoy playing the game of their choice and see their highscore.

Functionalities such as game rating will also be available, as well as an option to rate and write a short reviews for the game they have played.

The page makes use the most out of the following technologies:
  • Marvel Comics API: for requesting data and images used in our mini-games.
  • PrimeNg and AngularMaterial: for designing the layout of the page, grid model, etc.

Highscore page

On this page the users can find information on the global scoreboard section that will let him know how well he is doing.
