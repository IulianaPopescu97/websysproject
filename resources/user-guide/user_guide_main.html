Case study - Authentication

MarvelHeroJam application is a gaming web platform intended for use by anyone wanting to develop their strategic thinking and hand-eye coordonation. On the website, players can enjoy a variety of dificulties of the tic-toc game involving their favorite Marvel characters. This selection of dificulties is intended to give people a relaxing and entertaing way of enjoing their free time with their beloved heroes bu plalying with a friend or an AI. These scenarios are meant to offer a clearer image of how to use the MarvelHeroJam application.

--------------------------------------------------------------------------------------------------------------------

Register user - Section 1

The registration form requires a valid email address of user's choice for their profile on the web application. The e-mail is only required for the validation of the user account. Password is necessary only for login into the application (Section 1). For safety reasons, the filled password needs to match the confirm password..

For the user to access the application, the email needs to be validated first.

Register user - Section 2

As it the can seen the register form, the user can also sign up with the google account. When the button "Continued with google" is pressed, a form for google form will be opened. The email validation is not required in this case.

Register user - Section 3

A button can be pressed so that the user can be re-directed back to the log in form.


--------------------------------------------------------------------------------------------------------------------

Sign in - Section 1

The login form requires the name and password of a registered user (Section 1). If the provided user data is correct, the user will be redirected to the Dashboard page.

Sign in - Section 2

If the user registered in the first part with a google account, in this section, the user can access the application by using the google account.

Sign in - Section 3

The user can use the change password functionality. If the user hovers the mouse to the "Forgot password" button, a message will be displayed. When the button is pressed, a new form will be opened. The email address linked to the account needs to be provided. If the email is correct an email will be sent to change the password.

Note: Help messages are integrated for a number of different buttons


--------------------------------------------------------------------------------------------------------------------

Once the user has registered and logged in with the correct credentials, they will have access to the application. They will be redirected to the Dashboard page and access the menu with all the functionalities.

The user can log out from the application by pressing the button "Log out" from the profile menu, on the top-right side.

Case study - Profile editing

User profile

Once the user is loged in, the menu is available in the left side. In the "User profile" page, the user can change the picture (Section 1) and to check if the email was verified (Section 2). The image will be displayed in the top-right side.

When the email is verified, a check button will be displayed in this page. When the mouse is hovered, a message will be displayed.

--------------------------------------------------------------------------------------------------------------------


Settings

In the "Settings" page, the user can edit some data information from his account.


For example: The user name can be changed. The user name will be displayed when a game is played. When the mouse cursor will be hovered on the name field, a message will be displayed to the user.

When the field is pressed, a form will be opened to edit the name. The initial text will be copied in the form as background.

Editing the email will also need the password

And for the password to be changed, the new and the old password needs to be provided.

Case study - Games

Games

On the "Games" page, different dificulties for the stategy game "Tic-tac-toe" can be selected.

The user name will be used to personalized the game page. When "Games" page is opened, two marvel characters will be randomly picked to play with. As it can be seedn in the image: "Blink VS Iron Man" When the cursor is put on the character name, some information will be displyaed to the user.


2Players

The game can be played in two persons. The image of the characters will be displayed on the conventional "X" and "0".

In order to win the game, a player must place three of their marks in a horizontal, vertical, or diagonal row.

The game can be reseted or press the "Start a new game" to go back to select e different difficulty.


1 Player Easy

The game will be played by the user and the computer. In this level, the computer can be easily beaten.

Spoiler: the computer will select the next empty cell.


1 Player Medium

The game will be played by the user and the computer. In this level, the computer is just a little harder to beat.

Spoiler: the computer will select the empty cells and will put one random.

1 Player Expert

The game will be played by the user and the computer. In this level, the computer will be very hard to beeat. The user will find himself most of the times in a draw. As "Tic-tac-toe" is fairly simple game, it can be used in the branch of artificial intelligence that deals with the searching of game trees.

Spoiler: The algorithm used in here is Minimax.

AI moves first

To user can also choose for the AI to start the game. This setting will be applicable only for the 1 player based games.
